Welcome to See The Code

Important: Use a Web Server

This demo must be served through a web server - opening the HTML file directly (file://) will not work due to CORS restrictions.

Quick Server Options:

  • Python 3: python -m http.server 8000 (then visit http://localhost:8000/demo/)
  • Node.js: npx http-server -p 8000 (then visit http://localhost:8000/demo/)
  • VS Code: Use the "Live Server" extension
  • PHP: php -S localhost:8000

How to Test:

  1. Start a web server in the project root directory (see options above)
  2. Open http://localhost:8000/demo/ in your browser
  3. Open your browser's developer console (F12)
  4. Inject the overlay script: 
    // Set configuration BEFORE loading the script
window.SeeTheCodeConfig = {
  codeMapUrl: './code-map.json',  // Path to code map relative to current page
  // workspaceRoot is optional - only needed for VS Code file opening
  // If your code-map.json uses relative paths, set this to your project root:
  // workspaceRoot: '/path/to/your/project'  // Windows: 'E:/path/to/project' or Mac/Linux: '/Users/name/project'
};

// Inject the overlay script
const script = document.createElement('script');
script.src = '../overlay/inject.js';  // Go up one level from demo/ to project root
script.onload = () => {
  console.log('Overlay loaded successfully');
  if (window.SeeTheCode) {
    console.log('SeeTheCode API is ready');
  }
};
script.onerror = () => {
  console.error('Failed to load script. Make sure you are using a web server!');
  console.error('Current URL:', window.location.href);
  console.error('Tried to load from:', script.src);
};
document.head.appendChild(script);

// Optional: Set workspace root after loading (if needed for VS Code)
// window.SeeTheCode.setWorkspaceRoot('/path/to/your/project');
  5. Look for "See the code" badges on elements that match the code-map.json
  6. Click badges to see file and line information
  7. Enable debug mode in console: 
    window.SeeTheCode.setDebug(true);

Debug Mode

Enable debug mode to see unmatched selectors highlighted in red. This helps identify elements that don't have mappings in code-map.json.

Run this in the console:

window.SeeTheCode.setDebug(true);

To see statistics:

window.SeeTheCode.getStats();
